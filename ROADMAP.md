# Ohm Neuro Intelligence — Build Report

**Stack:** React 19 · Vite · Supabase · Vercel · Anthropic Claude API · PubMed (NCBI) API  
**Last updated:** 3 May 2026  
**Live URL:** https://cmgt.vercel.app

---

## System Architecture

```
PubMed API (NCBI)
  → 15 neuroscience search queries
  → scored by recency + publication type
  → best paper per category selected
       ↓
Triage history (Supabase reactions table)
  → boosts categories editors have drafted
  → skips categories excluded 3+ times
  → injects past drafted/excluded titles into Claude context
       ↓
Claude Sonnet 4.6
  → editorial title, brief, key claims, signal summary
  → source_url taken from PubMed DOI — Claude cannot modify it
       ↓
Supabase (topic_cards table)
  → React feed (Intelligence Feed view)
       ↓
Editor triages: Draft / Monitor / Exclude
       ↓
Make.com webhook → Google Doc created → draft_doc_url written back to Supabase
```

## Completed Features

### 1. Real Paper Sourcing — No More Hallucination
- Every topic card is backed by a **real, published PubMed paper**
- The system queries PubMed across 15 neuroscience topic areas per batch
- Each query returns up to 5 results; the best paper is selected by scoring:
  - Recency: 2023+ (+3), 2021+ (+2), 2019+ (+1)
  - Publication type: Randomized controlled trial (+3), Meta-analysis (+2), Journal article (+1)
  - Must have both an abstract and a DOI to qualify
- `source_url` on every card is a verified `https://doi.org/...` link — never generated by Claude

### 2. Triage-Aware Query Ranking
- Categories editors have frequently **drafted** get their PubMed queries run first
- Categories excluded **3 or more times** are skipped entirely from the query set
- The system learns editorial preferences and adjusts what papers it fetches each batch — not just what Claude writes about

### 3. Claude Editorial Layer
- Claude receives real paper abstracts and writes:
  - Reader-friendly editorial title (not the raw paper title)
  - 2–3 sentence brief on the core finding and brain health relevance
  - 3 key claims with supporting statistics from the paper
  - Signal summary: `FULL PIECE` or `SUPPORTING REFERENCE` — one sentence on why it matters
- Dynamic context is prepended from the triage history: past drafted topics (up to 12), soft-yes topics (up to 8), excluded topics (up to 15) — Claude uses these to calibrate tone, depth, and topic avoidance

### 4. Token Efficiency & Cost Control
- Abstracts truncated to 450 characters (sufficient for editorial signals, not full text)
- Dynamic context capped at 35 entries max — no unbounded growth over time
- "Do Not Repeat" section removed from Claude prompt — title deduplication happens at PubMed fetch level instead, saving ~200 tokens per run
- One unused Supabase query removed from context builder
- Estimated cost: **~$0.03 per Regenerate run** (combined input + output at Claude Sonnet pricing)

### 5. Generation Progress Indicator
- Replaces the blank "Loading…" screen during Regenerate with a 4-step progress tracker:
  1. Checking your triage history
  2. Querying PubMed for real papers
  3. Claude writing editorial cards
  4. Saving cards to database
- Each step shows a pulsing dot while active, checkmark when done
- Progress bar + elapsed seconds displayed below
- Accurate timing: 60–90 seconds total (pipeline is intentionally thorough)

### 6. Triage System (3-Action Model)
- **Draft** — opens notes panel, fires Make.com webhook to create Google Doc, moves card to Pipeline
- **Monitor** — one-click, marks for soft tracking, no panel required
- **Exclude** — opens qualifier panel with reason selection (Already covered, Off-narrative, Weak evidence, Too niche, Not relevant to audience)
- All three are undoable
- All reactions stored in `reactions` table with timestamp and optional notes

### 7. Google Docs Integration (Make.com)
- Drafting a card fires a webhook to Make.com
- Make creates a Google Doc from template and populates: title, brief, notes, key claims, signal summary, citation, source URL, category
- Doc is moved to OHM Neuro Drafts Google Drive folder
- `draft_doc_url` is written back to Supabase
- "Open Draft ↗" link appears on the card in both the feed and Pipeline view

### 8. Pipeline View
- Shows all drafted topic cards, newest first
- Expandable cards: click to reveal key claims (numbered list), signal summary (highlighted), source type badges, Source ↗ link, Open Draft ↗ button
- Stat bar: total drafted + drafted this week

### 9. Prompt Inspector
- Two tabs: **PubMed Queries** (all 15 active search queries + scoring explanation) and **Claude Prompt** (full assembled system + user prompt saved on every run)
- Timestamps each run, shows version label and change source
- Character count + estimated token count shown per prompt

### 10. Collapsible Sidebar
- Desktop: sidebar collapses to hidden with a small re-expand tab at the left edge
- Mobile: sidebar is a slide-in overlay with backdrop dismiss
- Hamburger button toggles collapse on desktop, opens overlay on mobile

### 11. Mobile Interface
- Compact header with stats displayed as inline pills (not 3-column grid)
- Regenerate button in the top bar on mobile — icon-only to save space
- Touch targets at 36px minimum
- Tighter padding throughout

### 12. Infrastructure & Reliability
- Supabase-backed rate limiting on Regenerate (survives Vercel cold starts)
- Vercel function timeout set to 120 seconds — safely covers the 60–90s pipeline
- Duplicate DOI prevention: last 100 source URLs are pre-fetched and filtered before PubMed selects papers — the same paper can never appear twice
- Card validation accepts 5–10 cards (was exact-count only, causing silent failures when Claude returned 9 instead of 10)
- Row-by-row fallback if bulk insert fails — partial batches still saved
- RLS policy allowing anon key to update `topic_cards.feed_status`

- Debug console.logs removed from production build
